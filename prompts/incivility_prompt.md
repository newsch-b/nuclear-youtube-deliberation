# 不文明留言標記提示詞（Incivility Annotation Prompt）

**模型**：GPT-4.1-mini  
**任務**：對全部 223,691 筆有效留言進行不文明的二元標注（0 = 文明，1 = 不文明）  
**定義基礎**：Muddiman et al.（2018）——不文明 = 違反「禮貌規範」**或**「民主規範」的言論

---

## Prompt

```
你是一位專門研究台灣社群媒體與政治溝通的內容標記研究員。

【任務】
以下是台灣核能主題的YouTube影片留言，請獨立判斷每則留言是否包含「不文明言論」，
請務必先在 reason 欄位進行邏輯分析，後才決定 label。

【不文明言論判定標準】
只要出現以下任一情況，即標記為 1：

*違反禮貌規範*
辱罵或羞辱、粗俗用語、髒話、諷刺或反諷、人身攻擊、負面標籤稱呼、
輕蔑語氣、貶低特定對象或群體、質疑智力、攻擊名譽

*違反民主規範*
煽動暴力、挑釁、歧視、威脅、宣稱政府無能、體制崩壞、治理失敗

否則標記為 0。

【輸出要求】
- 僅輸出 JSON
- 不得新增欄位

格式範例：
{
  "results": [
    {
      "id": "留言原始ID",
      "reason": "在此先寫下判定理由（20字內）",
      "label": 0或1
    }
  ]
}
```

---

## 設計說明

### 理由先行（Chain-of-thought lite）
要求模型先在 `reason` 欄位寫出判定邏輯，再決定 `label`，可降低邊界案例的誤判率。

### 批次處理
每次 API 呼叫可傳入多筆留言（建議 10–20 筆），model 回傳 `results` 陣列。

### 標注策略比較（信度）

不同標注策略的信度比較（以 κ ≥ 0.61 為可接受門檻）：

| 策略 | Cohen's κ |
|------|-----------|
| 最佳策略（含 reason 欄位） | 0.701 |
| 無 reason 欄位 | 較低 |

## 操作型定義補充

詳細範例請見 `codebooks/incivility_codebook.md`。

### 常見不文明模式（台灣核能留言）

| 類型 | 範例模式 | 屬於 |
|------|----------|------|
| 政黨標籤化貶稱 | 「冥禁黨」、「狗民黨」 | 違反禮貌規範 |
| 人身攻擊 | 針對特定政治人物的侮辱 | 違反禮貌規範 |
| 威脅或煽動 | 「應該去死」、「滾出台灣」 | 違反民主規範 |
| 治理失敗宣稱 | 「政府根本無能」 | 違反民主規範 |

### 邊界案例處理

- **諷刺或反諷**：即便不含髒話，仍標記為 1（違反禮貌規範）
- **強烈批評政策**（但無人身攻擊）：標記為 0，除非同時含「治理失敗」宣稱
- **「政府失職」等一般性評論**：需判斷語氣強度，含蔑視語氣者標記為 1
